
## 第一步：地图 → `map.json`

### 完整 Schema

```json
{
  "metadata": {
    "title": "string",           // 地图标题
    "description": "string",     // 地图概述
    "total_locations": number,   // 地点总数
    "floors": ["string"]         // 层级/区域列表，如 ["村庄", "森林", "地下"]
  },
  "start_location_id": "loc_001",
  "locations": [
    {
      "location_id": "loc_001",          // 唯一ID，格式 loc_XXX
      "name": "string",                  // 地点名称
      "description": "string",           // 环境描述（2-4句，有画面感）
      "floor": "string",                 // 所属层级，对应 floors 中的值
      "danger_level": "string",          // 危险等级：safe | low | medium | high | extreme
      "connections": [
        {
          "target_location_id": "loc_XXX",   // 目标地点ID
          "connection_type": "string",        // 连接类型：path | door | stairs | elevator | vent | tunnel | hidden
          "bidirectional": true,              // 是否双向通行
          "locked": false,                    // 是否需要解锁
          "lock_condition": {                 // 解锁条件（locked=true时必填）
            "type": "item | flag | attribute",
            "name": "string",                 // 条件名称，如 item_id 或 flag 名
            "value": "any"
          }
        }
      ],
      "npcs": ["npc_XXX"],       // 该地点出现的NPC ID列表
      "items": ["item_XXX"],     // 该地点可获取的物品ID列表
      "events": ["string"]       // 可能触发的特殊事件描述
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `location_id` | string | 唯一标识，格式 `loc_001`、`loc_002` |
| `floor` | string | 所属层级，必须是 `floors` 数组中的值 |
| `danger_level` | enum | `safe` / `low` / `medium` / `high` / `extreme` |
| `connection_type` | enum | `path` / `door` / `stairs` / `elevator` / `vent` / `tunnel` / `hidden` |
| `locked` | boolean | `true` 时必须填写 `lock_condition` |
| `npcs` | string[] | 引用 `npcs.json` 中的 `npc_id` |
| `items` | string[] | 引用 `items.json` 中的 `item_id` |

### 生成规则

1. 每个地点至少与 1 个其他地点相连
2. 起始地点 `start_location_id` 必须存在于 `locations` 中
3. 锁定的连接必须有明确的 `lock_condition`
4. 地图应包含至少 1 个 `safe` 区和至少 1 个 `high`/`extreme` 区
5. 隐藏连接（`hidden`）用于密道、暗门等

## 第二步： NPC → `npcs.json`

### 完整 Schema

```json
{
  "metadata": {
    "title": "string",
    "description": "string",
    "total_npcs": number
  },
  "characters": [
    {
      "npc_id": "npc_001",              // 唯一ID，格式 npc_XXX
      "name": "string",                 // NPC名称
      "type": "string",                 // 类型：human | cultist | creature | mythos | machine | other
      "description": "string",          // 外貌/特征描述
      "personality": "string",          // 性格/行为模式（null 表示无意识生物）
      "danger_level": "string",         // 危险等级：safe | low | medium | high | extreme
      "hostility": "string",            // 敌意：friendly | neutral | hostile | conditional
      "spawn_locations": ["loc_XXX"],   // 初始出现位置
      "interactions": [
        {
          "type": "string",             // 交互类型：talk | trade | fight | avoid | help | persuade | intimidate | pray
          "result": "string"            // 交互结果描述
        }
      ],
      "drops": ["item_XXX"],            // 可掉落/获得的物品ID
      "special_abilities": ["string"],  // 特殊能力（生物/SCP专用）
      "weakness": ["string"],           // 弱点
      "lore": "string"                  // 背景故事
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `npc_id` | string | 唯一标识，格式 `npc_001`、`npc_002` |
| `type` | enum | `human` / `cultist` / `creature` / `mythos` / `machine` / `other` |
| `hostility` | enum | `friendly` / `neutral` / `hostile` / `conditional` |
| `personality` | string\|null | 人类NPC填写性格描述，无意识生物填 `null` |
| `spawn_locations` | string[] | 引用 `map.json` 中的 `location_id` |
| `drops` | string[] | 引用 `items.json` 中的 `item_id` |

### NPC 类型说明

| 类型 | 适用场景 |
|------|---------|
| `human` | 普通人类村民、研究员、警卫等 |
| `cultist` | 邪教徒、被洗脑的人类 |
| `creature` | 普通生物、变异体 |
| `mythos` | 克苏鲁神话生物、SCP实体 |
| `machine` | 机械/AI |
| `other` | 其他异常存在 |

## 第三步：物品 → `items.json`

### 完整 Schema

```json
{
  "metadata": {
    "title": "string",
    "description": "string",
    "total_items": number
  },
  "objects": [
    {
      "item_id": "item_001",            // 唯一ID，格式 item_XXX
      "name": "string",                 // 物品名称
      "type": "string",                 // 类型：key | weapon | tool | consumable | document | equipment | artifact | tome | scp_object
      "description": "string",          // 物品描述
      "rarity": "string",               // 稀有度：common | uncommon | rare | legendary | unique
      "weight": number,                 // 重量（kg）
      "stackable": false,               // 是否可堆叠
      "effects": [
        {
          "effect_type": "string",      // 效果类型：unlock | heal | damage | buff | sanity | knowledge | special
          "target": "string"            // 效果目标（unlock时填 loc_id；其他填属性名或描述）
        }
      ],
      "uses": number,                   // 使用次数（-1 表示无限）
      "spawn_locations": ["loc_XXX"],   // 可获取位置
      "dropped_by": ["npc_XXX"],        // 可掉落的NPC
      "required_for": ["string"]        // 用途说明（如"进入村长家地下室"）
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `item_id` | string | 唯一标识，格式 `item_001`、`item_002` |
| `type` | enum | `key` / `weapon` / `tool` / `consumable` / `document` / `equipment` / `artifact` / `tome` / `scp_object` |
| `rarity` | enum | `common` / `uncommon` / `rare` / `legendary` / `unique` |
| `uses` | number | `-1` 表示无限次使用；`1` 表示一次性消耗品 |
| `effects[].effect_type` | enum | `unlock` / `heal` / `damage` / `buff` / `sanity` / `knowledge` / `special` |
| `effects[].target` | string | `unlock` 时填 `loc_id`；其他填属性名或效果描述 |
| `spawn_locations` | string[] | 引用 `map.json` 中的 `location_id` |
| `dropped_by` | string[] | 引用 `npcs.json` 中的 `npc_id` |

### 物品类型说明

| 类型 | 说明 |
|------|------|
| `key` | 钥匙/钥匙卡/解锁道具 |
| `weapon` | 武器（枪、刀、棍棒等） |
| `tool` | 工具（手电筒、绳索、螺丝刀等） |
| `consumable` | 消耗品（急救包、药剂、食物等） |
| `document` | 文档（日记、档案、地图、经文等） |
| `equipment` | 装备（防护服、面具等） |
| `artifact` | 神器/异常物品（护身符、封印石等） |
| `tome` | 禁忌书籍（古籍、法典等，阅读会影响理智） |
| `scp_object` | SCP物品（现实锚、SCP-500等） |

## 第四步：剧本 → `script.json`

### 完整 Schema

```json
{
  "metadata": {
    "title": "string",              // 剧本标题
    "description": "string",        // 剧本简介
    "total_nodes": number,          // 节点总数
    "theme": "string",              // 主题：coc | scp | horror | mystery | survival
    "difficulty": "string",         // 难度：easy | normal | hard | extreme
    "required_items": ["item_XXX"], // 通关必需物品（可选）
    "required_knowledge": ["string"] // 通关必需知识/线索（可选）
  },
  "nodes": [
    {
      "node_id": "node_001",              // 唯一ID，格式 node_XXX
      "location_id": "loc_XXX",           // 关联地点，引用 map.json 中的 location_id
      "title": "string",                  // 节点标题
      "description": "string",            // 场景描述（2-4句，有画面感）
      "roll_required": false,             // 是否需要 Roll 点
      "roll_type": "string",              // Roll 类型（roll_required=true时必填）
      "roll_difficulty": "string",        // Roll 难度（roll_required=true时必填）
      "conditions": [                     // 进入本节点的前置条件（可为空数组）
        {
          "type": "string",               // 条件类型：attribute | flag | item | roll_result
          "name": "string",               // 条件名称
          "operator": "string",           // 运算符：>= | <= | == | > | < | exists | not_exists
          "value": "any"                  // 比较值
        }
      ],
      "outcomes": [                       // 分支出口（至少1个，结局节点可只有1个）
        {
          "description": "string",        // 分支描述（玩家看到的选项文字）
          "next_node": "string",          // 下一个节点ID（结局节点填 "node_end"）
          "target_location_id": "string", // 移动到的目标地点ID
          "items_gained": ["item_XXX"],   // 该分支获得的物品
          "npcs_encountered": ["npc_XXX"],// 该分支遭遇的NPC
          "condition": {                  // 触发条件（可为空对象 {}）
            "roll_success": true,         // Roll 成功/失败（roll_required=true时使用）
            "type": "string",             // 其他条件类型
            "name": "string",
            "operator": "string",
            "value": "any",
            "is_ending": true,            // 是否为结局（结局节点必填）
            "ending_type": "string",      // 结局类型：good | neutral | bad | sacrifice | sanity_collapse | hidden
            "ending_title": "string"      // 结局标题
          }
        }
      ]
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `node_id` | string | 唯一标识，格式 `node_001`、`node_002` |
| `location_id` | string | 引用 `map.json` 中的 `location_id` |
| `roll_type` | enum | `luck` / `strength` / `dexterity` / `intelligence` / `charisma` / `perception` / `san` / `none` |
| `roll_difficulty` | enum | `easy` / `medium` / `hard` / `extreme` / `critical` / `none` |
| `conditions[].type` | enum | `attribute` / `flag` / `item` / `roll_result` |
| `conditions[].operator` | enum | `>=` / `<=` / `==` / `>` / `<` / `exists` / `not_exists` |
| `outcomes[].condition.roll_success` | boolean | `true`=成功分支，`false`=失败分支 |
| `outcomes[].condition.is_ending` | boolean | `true` 表示该分支为结局 |
| `outcomes[].condition.ending_type` | enum | `good` / `neutral` / `bad` / `sacrifice` / `sanity_collapse` / `hidden` |

### Roll 类型说明

| Roll 类型 | 适用场景 |
|-----------|---------|
| `perception` | 发现隐藏线索、察觉危险 |
| `intelligence` | 解读古籍、分析情报 |
| `dexterity` | 逃跑、躲避、潜行 |
| `strength` | 战斗、破坏、体力挑战 |
| `charisma` | 说服NPC、社交 |
| `san` | 面对恐怖时的精神检定 |
| `luck` | 随机事件 |

## 数据关联规则

所有文件通过 ID 互相引用，必须保持一致：

```
map.json
  locations[].npcs[]      → 引用 npcs.json 中的 npc_id
  locations[].items[]     → 引用 items.json 中的 item_id
  connections[].lock_condition.name → 引用 item_id 或 flag 名

npcs.json
  characters[].spawn_locations[] → 引用 map.json 中的 location_id
  characters[].drops[]           → 引用 items.json 中的 item_id

items.json
  objects[].spawn_locations[]    → 引用 map.json 中的 location_id
  objects[].dropped_by[]         → 引用 npcs.json 中的 npc_id
  objects[].effects[].target     → unlock 时引用 map.json 中的 location_id

script.json
  nodes[].location_id            → 引用 map.json 中的 location_id
  nodes[].outcomes[].target_location_id → 引用 map.json 中的 location_id
  nodes[].outcomes[].items_gained[]     → 引用 items.json 中的 item_id
  nodes[].outcomes[].npcs_encountered[] → 引用 npcs.json 中的 npc_id
```
